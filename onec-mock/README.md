# Тестовая 1С:Предприятие (эмулятор)

Самостоятельный сервис, имитирующий REST-сервис учётной системы 1С.
Нужен, чтобы показать **сквозной обмен**: АИС сопровождения возвратов
отправляет данные → «1С» принимает их, создаёт учётные документы и
показывает их в веб-интерфейсе.

> Это **не настоящая 1С** и не повторяет её функционал (учёт, проводки,
> остатки). Сервис воспроизводит только тот REST-интерфейс, с которым
> взаимодействует АИС, и формирует наглядные печатные формы документов.

## Что принимает (как реальная 1С-публикация HTTP-сервиса)

| Метод | Путь | Документ 1С |
|---|---|---|
| POST | `/returns` | Возврат товаров от покупателя |
| POST | `/write-off` | Акт о списании товаров (ТОРГ-16) |
| POST | `/refund` | Возврат денежных средств покупателю |
| PUT  | `/stock` | Корректировка складских остатков |

(те же пути доступны и с префиксом `/api`).

## Запуск

```bash
cd onec-mock
docker compose up --build
```

Интерфейс: **http://localhost:8081** (список принятых документов, автообновление).

Без Docker:
```bash
pip install -r requirements.txt
uvicorn main:app --host 0.0.0.0 --port 8081
```

## Как подключить к АИС

АИС читает адрес 1С из переменной `ONEC_API_URL`. Так как бэкенд АИС
работает в Docker, до сервиса на хосте он достучится через
`host.docker.internal`.

В `app/docker-compose.yml` у сервиса `backend` укажите:

```yaml
    environment:
      ONEC_API_URL: http://host.docker.internal:8081
```

затем перезапустите бэкенд:

```bash
docker compose -f app/docker-compose.yml up -d backend
```

После этого выполните в АИС **финансовое завершение возврата** —
документ появится на http://localhost:8081.
